K-Wise Codex Security Hardening Report

Generated: 2026-06-07

Scope: repository-wide frontend, backend, kiosk routes, admin routes, database-facing services, dependency manifests, and live localhost smoke checks.

Baseline

Controls were mapped to OWASP API Security Top 10 2023, OWASP CSRF, OWASP Logging, Express production security, and deterministic PC compatibility kiosk patterns.

Fixed Findings

  • Frontend localStorage auth trust: admin identity now comes from /api/auth/me; legacy auth storage is cleared; Bearer headers are stripped.
  • Credential transport and CSRF: login uses HttpOnly cookie auth plus CSRF token; mutating admin calls require CSRF.
  • Route exposure: admin-like mutation/stat/cache/history/settings/rule endpoints now require backend auth/RBAC.
  • Sensitive leaks: response/log sanitizers mask secrets and body logs were removed from kiosk order/upload hot paths.
  • Uploads/static files: magic-byte image validation, SVG rejection, random filenames, folder allowlists, size limits, cleanup, and safe static headers are enforced.
  • SQL hardening: spec-table identifiers are allowlisted and pagination/search inputs are capped.
  • Performance-safe security: pool defaults are capped outside load-test mode, rate limits are tightened, health checks avoid self-DoS, and canceled probes are silent.

  • Backend npm audit --omit=dev: 0 vulnerabilities.
  • Compatibility audit: 343ms, 312 files, 133 tables, 3,200 enabled rules, aiEnabled:false.
  • Browser smoke: kiosk/admin/legacy AI redirect routes had no fresh console errors; warm page loads were under 2s.
